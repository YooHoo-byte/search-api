Configuration Guide

Prerequisites

Before installing the Search API, ensure your system meets the following requirements:

Node.js

Version: 18.0.0 or higher

Check version:

node --version

npm or yarn

Package manager for dependencies

Check version:

npm --version
# or
yarn --version

System Resources

RAM: 512MB minimum (1GB recommended)

Disk: 100MB free space

Network: Internet connection

Terminal Access

Command line access for installation

Basic knowledge of terminal commands

Permission to install packages

Tip: If you don't have Node.js installed, download it from nodejs.org or use a version manager like nvm.

Installation

Step 1: Clone the Repository

Clone the Search API repository to your local machine:

# Clone using HTTPS
git clone https://github.com/YooHoo-byte/search-api.git

# Or clone using SSH
git clone git@github.com:YooHoo-byte/search-api.git

# Navigate to the project directory
cd search-api

Step 2: Install Dependencies

Install all required Node.js dependencies:

npm
yarn
npm install
yarn install

The installation will download and install the following dependencies:

  • express - Web server framework
  • cheerio - HTML parsing
  • node-fetch - HTTP requests
  • cors - Cross-origin resource sharing
  • helmet - Security headers
  • dotenv - Environment variable management

Configuration

Step 1: Create Environment File

Copy the example environment file and customize it for your setup:

# Copy the example environment file
cp .env.example .env

Step 2: Configure Environment Variables

Edit the .env file with your preferred text editor:

# Using nano
nano .env

# Using vim
vim .env

# Using Visual Studio Code
code .env

Configure the essential settings:

# Basic Configuration
PORT=3000
NODE_ENV=development
CORS_ORIGIN=*

# Search Providers
ENABLE_DUCKDUCKGO=true
ENABLE_STARTPAGE=true
ENABLE_IMAGES=true

# Safe Search
DEFAULT_SAFE_SEARCH=moderate

# Performance
REQUEST_TIMEOUT=20000
MAX_RETRIES=5

Important: Never commit your .env file to version control. It contains sensitive configuration details.

Running the Server

Development Mode

For development with auto-restart on file changes:

npm run dev

Production Mode

For production deployment:

npm start

Using PM2 (Production)

For production with process management:

# Install PM2 globally
npm install -g pm2

# Start the application
pm2 start src/index.js --name "search-api"

# Monitor the application
pm2 monit

# View logs
pm2 logs search-api

Expected Output

When the server starts successfully, you should see:

> search-api@3.2.0 start
> node src/index.js

Search API Server started on port 3000
Environment: development
Cache initialized
Providers: DuckDuckGo, StartPage, Images
Ready to accept requests...

Docker Installation

Using Docker Compose (Recommended)

Create a docker-compose.yml file:

version: '3.8'
services:
  search-api:
    image: search-api:latest
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - PORT=3000
      - CORS_ORIGIN=*
    restart: unless-stopped
    volumes:
      - ./data:/app/data

Start the container:

# Build and start
docker-compose up -d

# View logs
docker-compose logs -f

# Stop the container
docker-compose down

Manual Docker Build

# Build the image
docker build -t search-api .

# Run the container
docker run -p 3000:3000 \
  -e PORT=3000 \
  -e NODE_ENV=production \
  --name search-api \
  search-api

Verification

Test the API

Verify that the API is running correctly:

1. Check Server Status

curl http://localhost:3000/status

Expected response:

{
  "status": "online",
  "version": "3.2.0",
  "timestamp": "2025-12-24T10:30:00.000Z"
}

2. Test Search Endpoint

curl "http://localhost:3000/search?q=test"

Should return search results in JSON format.

3. Test Cache Status

curl http://localhost:3000/cache/stats

Should return cache statistics.

Web Browser Test

Open your web browser and navigate to:

  • http://localhost:3000/status
  • http://localhost:3000/search?q=hello+world

Troubleshooting

Port Already in Use

Error: Error: listen EADDRINUSE: address already in use :::3000

Solution:

# Change port in .env file
PORT=3001

# Or kill process using port 3000
sudo lsof -ti:3000 | xargs kill -9

Dependency Installation Failed

Error: npm ERR! Cannot find module

Solution:

# Clear npm cache
npm cache clean --force

# Delete node_modules and package-lock.json
rm -rf node_modules package-lock.json

# Reinstall dependencies
npm install

Docker Container Won't Start

Error: Container exits immediately

Solution:

# Check container logs
docker logs search-api

# Run with interactive mode to see errors
docker run -it --rm search-api

No Search Results

Issue: API returns empty results

Solution:

# Check provider status
curl http://localhost:3000/status

# Enable debug logging in .env
LOG_LEVEL=debug
DEBUG_PROVIDERS=true
